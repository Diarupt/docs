Interaction Events are events that are triggered when interaction with and AI. These event help you manage state and other workflows in your application.

See [Connect](/sdks/javascript/connect) on how to handle these events

## Event Types

### Open

<ParamField body="event" type="open" required>
    This event is triggered when the Interaction connection is opened and ready to send and receive data
</ParamField>

<ParamField body="data" type="string" required>
    This is a string message, e.g "data channel opened"
</ParamField>

### User Started Talking 
(Hands-on mode event)

<ParamField body="event" type="user-started-talking" required>
    This event is triggered when the user starts talking
</ParamField>

<ParamField body="data" type="null" required>
    No data is returned
</ParamField>

### User Stopped Talking
(Hands-on mode event)

<ParamField body="event" type="user-stopped-talking" required>
    Emitted after user stop talking plus the when handsOnTimeout elapses.
</ParamField>

<ParamField body="data" type="string" required>
    No data is returned
</ParamField>

### Close

<ParamField body="event" type="close" required>
    This event is triggered when the Interaction connection is close. This can happen for a number of reasons, including the user closing the browser tab or window, calling the [disconnect](/sdks/javascript/disconnect) function, or the connection being lost.
</ParamField>

<ParamField body="data" type="string" required>
    This is a string message, e.g "data channel closed"
</ParamField>

### Error

<ParamField body="event" type="error" required>
    This event is triggered when their is an error with the webrtc connection or any other error duting the interaction
</ParamField>

<ParamField body="data" type="Error" required>
    This return a javascript javascript Error object
</ParamField>
